# 测试与验收

## 测试层次

### 契约测试

- 每个项目文件都经过 Zod 校验。
- 节点注册表中的 kind、配置 Schema 和执行器保持对应。
- Worker 请求、进度、结果和错误消息均可解析。
- 为未来 `formatVersion` 变化保留迁移测试入口。

### 单元测试

- 图校验、拓扑排序和缓存失效。
- 每个转换节点的正常、空值和错误输入。
- CSV/JSON/XLSX 导入导出。
- 受限表达式和公式注入防护。
- 撤销重做和项目脏状态判断。

### 集成测试

- 文件适配器输出可以直接进入执行引擎。
- Worker 运行、取消和迟到消息隔离。
- 上游配置变化触发正确的下游重算。
- 项目保存后重新打开产生相同图和配置。

### 端到端测试

- Chromium、Firefox、WebKit 和 Pixel 7 模拟移动 Chromium 均执行工作区冒烟、CSV 导入/转换/导出和双输入 Join。
- Join E2E 从页面绑定两个 CSV，以 left/right 端口执行并验证 10 行和负责人字段。
- 项目文件、损坏输入、取消和配置错误由单元/组件或 Worker 集成测试覆盖；尚未为每一项建立浏览器 E2E。

## 固定数据集

- `orders-small.csv`：10 行，用于 E2E 和演示。
- `regions.json`：地区负责人，用于 Join。
- 公式注入、损坏文件和混合类型输入在测试代码内构造，不维护重复夹具。
- `orders-100k.csv`：性能测试运行时确定性生成 100,000 行、6 列和 3,171,749 B，不提交大文件。

测试数据必须是虚构数据，不使用公司真实业务数据或个人信息。

## 功能验收

- 所有 MVP 节点均可从 UI 配置并执行。
- 非法图和非法配置在执行前给出可定位错误。
- 导出的字段顺序和配置一致。
- 保存的项目文件不包含原始数据。
- 相同输入和项目文件重复运行得到相同结果。
- 核心演示不需要网络和后端服务。

## 性能验收

2026-07-12 当前开发机 Chromium 实测：

- 100,000 行、6 列 CSV 从文件注入到 1,000 行预览完成耗时 453 ms。
- JavaScript 堆增加 63,405,320 B（约 60.5 MiB）。
- 记录 1 个主线程长任务，155 ms。
- 自动回归阈值为 30 秒和 512 MiB 堆增量；预览只返回并窗口化渲染 1,000 行。

该结果不是标准硬件基线，也没有证明 20 列数据或同步解析期间可取消。后续结果必须记录设备、浏览器、数据集、测量方式和实际数值，不修改数字掩盖问题。

## 安全与隐私验收

- 源码中不存在 `eval` 和 `new Function` 执行用户表达式。
- CSV/XLSX 导出具有公式注入测试。
- 项目文件不能引用任意远程脚本。
- 文件内容不进入远程日志或错误上报。
- 错误详情不展示整份数据或超长单元格。
