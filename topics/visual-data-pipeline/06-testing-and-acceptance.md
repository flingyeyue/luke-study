# 测试与验收

## 测试层次

### 契约测试

- 每个项目文件都经过 Zod 校验。
- 节点注册表中的 kind、配置 Schema 和执行器保持对应。
- Worker 请求、进度、结果和错误消息均可解析。
- 为未来 `formatVersion` 变化保留迁移测试入口。

### 单元测试

- 图校验、拓扑排序和缓存失效。
- 每个转换节点的正常、空值和错误输入。
- CSV/JSON/XLSX 导入导出。
- 受限表达式和公式注入防护。
- 撤销重做和项目脏状态判断。

### 集成测试

- 文件适配器输出可以直接进入执行引擎。
- Worker 运行、取消和迟到消息隔离。
- 上游配置变化触发正确的下游重算。
- 项目保存后重新打开产生相同图和配置。

### 端到端测试

- 导入 CSV，转换并导出。
- 双输入 Join 和聚合。
- 配置错误定位到具体节点。
- 运行过程中取消。
- 打开损坏或不支持版本的项目文件。
- 刷新或新建前提示未保存修改。

## 固定数据集

- `orders-small.csv`：约 100 行，用于 E2E 和演示。
- `regions.json`：地区负责人，用于 Join。
- `mixed-types.csv`：空值、非法日期、不同数字格式。
- `formula-injection.csv`：以公式字符开头的文本。
- `orders-100k.csv`：由脚本确定性生成，只用于性能测试，不提交大文件时应提交生成脚本。

测试数据必须是虚构数据，不使用公司真实业务数据或个人信息。

## 功能验收

- 所有 MVP 节点均可从 UI 配置并执行。
- 非法图和非法配置在执行前给出可定位错误。
- 导出的字段顺序和配置一致。
- 保存的项目文件不包含原始数据。
- 相同输入和项目文件重复运行得到相同结果。
- 核心演示不需要网络和后端服务。

## 性能验收

性能目标必须在确定目标设备和浏览器后记录实测值。初始目标：

- 10 万行、20 列 CSV 可以完成导入和基础转换。
- 执行期间命令栏和取消按钮保持可交互。
- 数据预览不一次渲染全部行。
- 连续运行不会无限增长缓存或 Worker 数量。

如果目标未达到，记录设备、浏览器、数据集、测量方式和实际结果，不修改数字掩盖问题。

## 安全与隐私验收

- 源码中不存在 `eval` 和 `new Function` 执行用户表达式。
- CSV/XLSX 导出具有公式注入测试。
- 项目文件不能引用任意远程脚本。
- 文件内容不进入远程日志或错误上报。
- 错误详情不展示整份数据或超长单元格。

## 交付命令

具体命令在项目初始化后确认，预期至少包括：

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm test:e2e
pnpm build
```

最终验收必须记录真实退出结果和失败项，不能仅记录“已测试”。
