# 架构决策记录

## ADR-001：使用纯浏览器架构

- 状态：已接受
- 日期：2026-07-11
- 决策：MVP 不使用后端和数据库服务，数据在浏览器本地处理。
- 原因：符合项目约束，降低部署复杂度，并强化隐私和离线演示能力。
- 代价：受浏览器内存限制，不能进行服务器定时任务、多人协作或超大规模计算。

## ADR-002：项目文件不嵌入原始数据

- 状态：已接受
- 日期：2026-07-11
- 决策：`.pipeline.json` 只保存图、节点配置和元数据，重新打开时由用户重新选择源文件。
- 原因：避免项目文件巨大、泄露业务数据或产生重复存储。
- 代价：项目文件不能单独复现运行，演示和测试需要配套固定输入文件。

## ADR-003：执行引擎与 React 解耦

- 状态：已接受
- 日期：2026-07-11
- 决策：核心引擎作为纯 TypeScript 包，通过 Worker 协议被 UI 调用。
- 原因：提高可测试性、性能隔离和未来 CLI 复用能力。
- 代价：需要维护序列化 DTO，不能跨 Worker 直接传递第三方复杂对象。

## ADR-004：MVP 流水线必须是有向无环图

- 状态：已接受
- 日期：2026-07-11
- 决策：拒绝包含环的项目图。
- 原因：简化执行计划、缓存失效和错误恢复，并避免隐式无限循环。
- 代价：暂不支持迭代类数据处理。

## ADR-005：禁止任意代码执行

- 状态：已接受
- 日期：2026-07-11
- 决策：筛选和计算字段使用受限表达式，禁止 `eval`、`new Function` 和用户 JavaScript。
- 原因：导入的项目文件不可信，任意代码执行风险不可接受。
- 代价：高级用户可用的表达能力有限，需要明确支持的函数集合。

## ADR-006：M0 锁定 TypeScript 5.9.3

- 状态：已接受
- 日期：2026-07-11
- 决策：M0 使用 TypeScript 5.9.3，不采用 npm `latest` 指向的 TypeScript 7.x。
- 原因：当前 `typescript-eslint` 声明的 TypeScript peer range 小于 6.1，使用 7.x 会引入未验证兼容性；5.9.3 满足 strict 类型检查和现有工具链需求。
- 代价：暂不使用 TypeScript 7 的新能力；只有在 lint、构建和依赖兼容性得到验证后才升级。

## ADR-007：M0 Worker 只实现协议骨架

- 状态：已接受
- 日期：2026-07-11
- 决策：M0 Worker 实现消息校验、ping、空流水线 run、幂等 cancel 和结构化未实现错误，不实现数据节点执行。
- 原因：先冻结跨线程协议，再由独立引擎任务实现执行语义，避免 UI Worker 与执行引擎耦合。
- 代价：M0 页面不能处理真实文件；非空流水线会返回明确的 `INTERNAL_EXECUTION_ERROR` 诊断。
