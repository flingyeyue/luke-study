# 任务看板

## 状态定义

- `BLOCKED`：依赖尚未满足。
- `READY`：可以领取。
- `IN_PROGRESS`：已有终端负责。
- `REVIEW`：已提交，等待主终端验证。
- `DONE`：已合并且验证通过。

任务状态只有在真实进展发生后更新。负责人填写终端编号或分支名。

## 终端所有权

| 终端 | 分支 | 主要所有权 | 禁止直接修改 |
|---|---|---|---|
| 0 主协调 | `main` | workspace、contracts、集成、知识库 | 无 |
| 1 画布 | `feature/canvas` | `apps/web/src/features/canvas` | engine、file-adapters |
| 2 引擎 | `feature/engine` | `packages/pipeline-engine` | Web UI、文件适配器 |
| 3 文件 | `feature/file-adapters` | `packages/file-adapters`、项目文件 UI | engine 算法、画布 |
| 4 节点 UI | `feature/node-config` | 节点配置、预览、运行面板 | contracts、Worker 内核 |
| 5 QA 交付 | `feature/qa-delivery` | E2E、性能、安全、构建检查 | 产品代码，除非修复已确认缺陷 |
